Generator

  • A generator is a function that returns an iterator object and uses the yield obj instead of return the actual value.
g = (2**x for i in range(5)) # Generator
# not all values are created initially, 
# when `next' is called, the next one 
# can generate.

# This feature is called Lazy computation. 

for i in g: 
    print(i) # Generates an iterator from generator

g = [2**x for i in range(5)] # List
  • Advantage: less mems

The yield syntax. Any func with yield is a generator. aka. switching the execution flows between functions.

Similar with stackless thread in C++22.

def foo(): 
    yield 1 # <-- The first time 
    yield 1
    yield 4
    yield 5
    yield 1
    yield 4 # <-- The last time

g = foo()
next(g)

yield records the current point and temporarily returns the value that yield holds.

Example. Yielding in the arraylist

def __iter__(self): 
    for i in range(len(list.data)): 
        yield self.data[i]
