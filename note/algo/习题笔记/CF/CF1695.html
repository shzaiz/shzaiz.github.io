CF1695C

描述: 给你一个 $n \times m$ ($1 \le n, m \le 1000$) 的格点图，每个格子的值要么是 $-1$，要么是 $1$，现在问你，是否有一条从 $(1, 1)$ 到 $(n, m)$ 的路径，使得路径上经过的格点的值的和为 $0$。在路径中，只能从 $a_{i, j}$ 移动到 $a_{i + 1, j}$ 或是 $a_{i, j + 1}$（向右或是向下走）。

考虑DP: $f[i][j][k]$表示在$(i,j)$, 取得了$k$个1, 是否存在方案. 因为$k$非常的小, 可以用bitset艹过去.

int a[1005][1005];
bitset< 1001> f[1001][1001];
void solve(){
    int n, m; cin>>n>>m;
    F(i, 1, n) F(j, 1, m){
        cin>>a[i][j];
        f[i][j].reset();
        if(a[i][j] == -1) a[i][j] = 0;
    }

    if((n+m-1 )%2) {printf("No\n"); return ;}

    f[0][1][0] = 1;
    F(i, 1, n){
        F(j, 1, m){
            f[i][j] = (f[i-1][j] | f[i][j-1])<<a[i][j];
        }
    }
    printf("%s\n", f[n][m][(n+m)>>1]?"Yes":"No");


}
