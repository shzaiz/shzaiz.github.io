FHQTreap基本操作

发表于 2020-11-11 | 分类于 总结

其实, FHQ Treap是一个比较简单的平衡树.

$\Large\text{Caution:}$ 结构确定是Treap的本质.

原因:

  • 这棵Treap的根节点一定是 k 值最小的, 且它的左子树的 val 小于它的 val , 它的右子树的 val 大于它的 val .
  • 当我们确定根节点后, 它的左右子树的 val 的取值范围就确定了. 接下来发现, 它的左子树的根节点一定是左子树中 k 值最小的, 这个点也就是大Treap根节点的左儿子.
  • 右儿子同理, 一定是右子树中 k 值最小的, 我们会发现, 对于每一个节点都是如此, 那么一棵树所有的节点的 val 和 k 是确定的, 结构就是确定的.

About Treap

treap = tree + heap, 是一种树和堆组合形成的数据结构. 其中, 每个节点上储存一个关键值,key. 满足:

  • 二叉搜索树性质
  • 父节点的关键值 ≥ 儿子的关键值.

今天所说的FHQ-Treap 是一种不带旋转的Treap.中序遍历后的 值 序列是单调不减的

Split

函数split(o,val,&x,&y)表示将o节点按照参数val分成x,y两个子树. 返回编号.

Split According to Node value

按权值分裂,一棵节点的参数都小于等于给定参数, 另一棵大于给定参数.

二叉搜索树左小右大的性质, 我们知道:

  • 比较根节点权值 与 给定权值的大小关系
    • 若小于给定权值, 则根及左子树点都小于等于给定权值, 将其归入左树中.
      向右子树递归, 继续分裂子树.
    • 否则, 根及右子树点都大于等于给定权值, 将其归入右树中.
      向左子树递归, 继续分裂子树.
  • 直到叶子节点

Split according to Rank&size

注意到一个点在中序遍历中的输出顺序就是它的排名. 因此一个点的排名就是中序遍历中在它之前输出的点的个数. 换句话说,就是如果我们要查找一个排名为$k$的点, 应该从根节点遍历:

  • 如果k=t[t[o].ls].sz+1, 那么可以返回.
  • 如果t<=t[t[o].ls].sz, 那么进入左子树查找.
  • 如果k>t[t[o].ls], 那么应该进入右子树. 并且k=k-t[t[o].ls].sz-1.

Merge

Merge操作可以保证两棵小Treap恢复成一棵大Treap.

同样的, 合并操作也需要同时满足二叉搜索树的性质和二叉堆的性质.

  • 二叉搜索树的性质: 合并后的中序遍历序列不变
    • 若两个节点$u$,$v$ 分别为左右区间树的根节点,那么 u及其子树要么在 v的左子树中, 要么 v及其子树在 u的右子树中

  • 保证二叉堆的性质:
