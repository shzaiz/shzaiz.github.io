A Game

Games

Game #1

甲乙两人面对若干堆石子，其中每一堆石子的数目可以任意确定。两人轮流按下列规则取走一些石子，游戏的规则如下：

  • 每一步应取走至少一枚石子；

  • 每一步只能从某一堆中取走部分或全部石子；

  • 如果谁无法按规则取子，谁就是输家。

Game #2

甲乙两人面对若干堆石子，其中每一堆石子的数目可以任意确定。两人轮流按下列规则取走一些石子，游戏的规则如下：

  • 每一步应取走至少一枚石子；

  • 甲乙双方事先约定一个数 m，并且每次取石子的数目不能超过 m 个；

  • 每一步只能从某一堆中取走部分或全部石子；

  • 如果谁无法按规则取子，谁就是输家。

## Properities of the game

注意到我们可以用一个\(n\)元组描述一个局面. 例如\(J(a_1,a_2,a_3,\cdots ,a_n)\). 并且知道改变这些的顺序, 仍然代表同一个局面.

如果初始局面只有一堆石子，则甲有必胜策略。 因为甲可以一次把这一堆石子全部取完，这样乙就无石子可取了。

如果初始局面有两堆石子, 并且二者的数量相等, 那么乙胜利. 乙可以按照这样的顺序取数:

  • 如果甲在一堆中取若干石子，乙便在另一堆中取同样数目的石子；
  • 石子总数一直在减少，最后必定是甲无石子可取。

总结. 也就是对于\(J(1)\), 甲有必胜策略, 初始局面\(J(3,3)\), 乙有必胜策略.

这里, 我们定义 局面的和 如下: 如果\(J({\Huge,}_{i=1}^n a_i) +J({\Huge,}_{i=1}^n b_i) = J({\Huge,}_{i=1}^n a_i,{\Huge,}_{i=1}^n b_i)\). 这里, $, $ 表示列举的意思. 和求和记号相似.

为什么要研究求和呢? 因为这样我们就可以分解一个局面. 对于局面\(A,B,S\), 如果\(S=A+B\),那么称局面 S 可以分解为“子局面”A 和 B。

这又有什么用? 我们发现,初始局面可以分成两个相同的“子局面”，则乙有必胜策略. 乙可以这样子做:

  • 设初始局面 S=A+A，想象有两个桌子，每个桌子上放一个 A 局面；
  • 若甲在一个桌子中取石子，则乙在另一个桌子中对称的取石子；
  • 根据对称性，在甲取了石子之后，乙总有石子可取；
  • 石子总数一直在减少，最后必定是甲无石子可取。

对于一个和局面\(S\), 若先行者有必胜策略, 称“S 胜”。否则叫做称“S 负”。

  • 例如$J(1) = $胜, \(J(3,3)=\)负.

如何利用局面的分解判断胜负?

首先我们查看两个性质:

  • 如果局面\(S\) 胜, 一定 存在 有\(S\to T\), 使得\(T\)负. 也就是这个人太菜了. 驾驭不了这个游戏.
  • 如果局面\(T\)负, 对于 任意 \(S\to T\), 使得\(T\) 胜. 也就是如果你先手失败, 不会再反败为胜了.(对称性)

然后, 我们设这个局面 S 可以分解成两个子局面 A 和 B.称一个石子也没有的局面为“空局面”。空局面是“负”局面。 分类讨论:

  • 若 A 和 B 一胜一负，则 S 胜。
    • 令A 胜 B 负. 并且有两个桌子 A 和 B，桌子上分别放着 A 局面和 B 局面；
    • 因为 定义 A 胜，所以甲可以保证取桌子 A 上的最后一个石子；
    • 因为 B 负，所以甲还可以保证取桌子 B 中的最后一个石子；
    • 综上所述，甲可以保证两个桌子上的最后一个石子都由自己取得。
  • 若A 负 B 负，则 S 负。
    • 无论甲先从 A 中取，还是先从 B 中取，都会变成一胜一负的局面；
    • 因此，乙面临的局面总是“胜”局面，故甲面临的 S 是“负”局面。
  • 若A胜 B 负，则 S 胜。
  • 若 A 胜 B 胜，则有时 S 胜，有时 S 负。
    • S=A+C+C，则 S 的胜负情况与 A 相同。
    • 令 B=C+C，则 S=A+B 且 B 负，故 S 的胜负情况与 A 相同。

局面的简化. 如果局面 S 中，存在两堆石子，它们的数目相等。用 T 表示从 S 中把这两堆 石子拿掉之后的局面，则称“S 可以简化为 T”。并且知道一个局面的胜负情况，与其简化后的局面相同. 不能化简的局面称作最简局面.

因此, 我们可以用集合来表示最简局面. 例如局面\(J(5,5,6,6,1)\Leftrightarrow J(1) \Leftrightarrow \{1\}\).

局面的胜负判断. 如果用二进制 1 和 0，分别表示一个局面的胜或负, 可以发现: 局面的加法，与二进制的加法有很多类似之处。

  • 若 A 和 B 相同，则 A+B 负；若 a 和 b 相等，则 a+b 为 0
  • 若 A 胜 B 负，则 A+B 胜；若 a≠0 且 b=0，则 a+b≠0
  • 若 B 胜 A 负，则 A+B 胜；若 b≠0 且 a=0，则 a+b≠0
  • 若 A 负 B 负，则 A+B 负；若 a=0 且 b=0，则 a+b=0
  • 若 A 胜 B 胜，则 A+B 有时胜，有时负

发现:如果用二进制数 s 来表示一个局面 S 的胜或负，S 胜则 s≠0，S 负则 s=0. 并且局面的加法，与二进制数的加法，性质完全相同。

用二进制数表示局面. 用符号#S，表示局面 S 所对应的二进制数。

  • 如果局面 S 只有一堆石子，则用这一堆石子数目所对应的二进制数来表示 S。
  • 若局面 S=A+B，则#S=#A+#B。

对于 游戏A 而言:

定义\(f\): 若局面 S 只有一堆石子，设 S={a1}，则 f(a1)=#S，即 f(a1)=#(a1)。

得到:设局面 S=(a1, a2, …, an)，即 S=(a1)+(a2)+…+(an)，则#S=f(a1)+f(a2)+…+f(an)。 并且对于局面 S，若#S=0，则 S 负；若#S≠0，则 S 胜。

对于 游戏B 而言:

定义\(f\)：f(x)=x mod (m+1)；把函数 f 的值看作是二进制数。

对于任意初始局面 S=(a1, a2, …, an)，令#S=f(a1)+f(a2)+…+f(an)。

发现, 游戏 B 的解法与游戏 A 十分类似。这是因为两个游戏的规则相当类似。

Game #3.

甲乙两人面对若干排石子，其中每一排石子的数目可以任意确定。

两人轮流按下列规则取走一些石子，游戏的规则如下：

  • 每一步必须从某一排中取走两枚石子；
  • 这两枚石子必须是紧紧挨着的；
  • 如果谁无法按规则取子，谁就是输家。

我们刚刚怎么做的?

设计一个函数 f，把函数 f 的值看作是二进制数。对于任意一个初始局面 S， 设 S=(a1, a2, …, an)，令#S=f(a1)+f(a2)+…+f(an)。若#S≠0，则先行者（甲）有 必胜策略；否则#S=0，这时后行者（乙）有必胜策略。

回忆: 函数 f 是否满足要求，关键在于#S 是否满足下面的条件:

  • 若#S≠0，则先行者必然存在一种取子方法 S→T，且#T=0。(胜利\(\to\) 失败)
  • 若#S=0，则无论先行者如何取子 S→T，都有#T≠0。(失败\(\cancel \to\)胜利)

用符号\(\sigma\)(x)，表示局面(x)的下一步所有可能出现的局面的集合。

  • 对于A游戏, \(\sigma (3)=[2,1,0]\).
  • 对于B游戏, 如果\(m=4\), \(\sigma(9)=[8,7,6,5], \sigma (2)=[1,0]\).
  • 对于C游戏, \(\sigma(7)=[[5],[1,4],[2,3]]\).

定义集合\(g(x)\), 设\(\sigma(x)=[{\Huge,}_{i=0}^kS_i]\), 那么\(g(x)=[{\Huge,}_{i=0}^k\ \#S_i]\).

令 \(\N^{+,0}\) 为非负整数全集，集合 G(x)表示集合 g(x)的补集。

定义函数 \(f(n)：f(n)=\min{G(n)}\)，即 \(f(n)\)等于集合$ G(n)$中的最小数。设局面 \(S=(a_1, a_2, …, a_n)\)，\(\#S=f(a_1)+f(a_2)+…+f(a_n)\)，采用二进制数的加法。若\(\#S=0\)，则 S 负；若\(\#S≠0\)，则$ S$ 胜。

Conclusion:

通用解法:

  • 用一个 n 元组\((a_1, a_2, …, a_n)\)，来描述游戏过程中的一个局面。
  • 用符号\(\#S\)，表示局面 \(S\) 所对应的二进制数。
  • 用符号\(\sigma(x)\)，表示局面\((x)\)的下一步所有可能出现的局面的集合。
  • 定义集合 \(g(x)\), 设\(\sigma(x)=[{S_1, S_2, …, S_k}]\)，则 \(g(x)=[{\#S_1, \#S_2, …, \#S_k}]\)。
  • 定义函数$ f(n)\(是\)g(x)$中没有出现的最小值.
  • 那么令\(\#S=f(a_1)+f(a_2)+…+f(a_n)\).
  • 若\(\#S≠0\)，则先行者有必胜策略；若\(\#S=0\)，则后行者有必胜策略。

适用范围:

  • 甲乙两人取石子游戏及其类似的游戏；
  • 每一步只能对某一堆石子进行操作；
  • 每一步操作的限制，只与这堆石子的数目或一些常数有关；
  • 操作在有限步内终止，并不会出现循环；
  • 谁无法继续操作，谁就是输家。
